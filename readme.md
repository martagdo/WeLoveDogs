# We Love Dogs 

## Descrição
Projeto de desenvolvimento frontend de um site institucional multipágina para o grupo social fictício **WeLoveDogs**, com foco na prática de HTML, CSS e responsividade.

## Funcionalidades
- Navegação entre páginas (Home, Quem Somos, Blog e Contato)
- Menu hamburger funcional para dispositivos móveis
- Layout responsivo
- Redirecionamento correto entre páginas
- Interface baseada em design do Figma

## Tecnologias
- HTML5
- CSS3
- JavaScript

## Layout
O projeto foi desenvolvido com base no seguinte protótipo:  https://www.figma.com/file/83KrYRp9nXE4wyMIWgxu6O/WeLoveDogs---N%C3%BAcleo-Web?node-id=0-1&t=1kBFqE3wLdtUsyxT-0

# Contexto
Projeto desenvolvido durante o programa de trainee da InfoJr, com o objetivo de consolidar conhecimentos em HTML, CSS e construção de interfaces web.

## Requisitos do desafio

Esta atividade tem duas partes principais: configuração do ambiente Git e desenvolvimento do site WeLoveDogs.

### Parte 1: Configuração do Git
Clone este repositório e crie uma branch própria para você (que futuramente servirá para o resto das atividades). O nome da sua branch deverá seguir o modelo `seu-nome`, por exemplo, se José for da turma ele deve nomear sua branch de `jose`.

### Parte 2: Desenvolvimento WeLoveDogs
O projeto WeLoveDogs tem por objetivo iniciar o aprendizado em HTML e CSS, 2 das 3 tecnologias bases do Front-End. O projeto possui o formato static multipage¹, com 4 páginas no total e a maneira de navegar e visualizar seu conteúdo por completo é através dos links do menu.

**Tarefa:** Com base no [mockup²](https://www.figma.com/file/83KrYRp9nXE4wyMIWgxu6O/WeLoveDogs---N%C3%BAcleo-Web?node-id=0-1&t=1kBFqE3wLdtUsyxT-0) apresentado, usando HTML e CSS, construa a webpage do grupo social WeLoveDogs. Você também deverá completar o jogo [FlexBox Froggy](https://flexboxfroggy.com/), isso ajudará a ter mais noção de como se organiza melhor os elementos de uma interface.

**Link do menu Hamburguer** https://www.figma.com/design/6ll9lLhSc8StjuKxq3xvdw/Trilha-Frontend-2025.2?node-id=0-1&t=hGzl3IIE26jlcOyf-1

---

*¹Static multipage. Consiste em um modelo da Web de um site com várias páginas, construídas usando HTML e seu conteúdo não muda independentemente da solicitação.*  
*²Mockup é uma representação estática de um projeto, bem próximo do design final.*

## Instruções para envio

1. **Clone** este repositório no seu computador
2. **Crie uma branch** a partir da main seguindo o padrão: `seu-nome` (ex: `jose`)
3. **Desenvolva a atividade** apenas nessa branch
4. **Faça commits constantes** com mensagens descritivas
5. **Suba regularmente** sua branch para acompanhamento
6. **Verifique cuidadosamente** se todos os requisitos foram cumpridos
7. **Faça um Merge Request** com o título: `WeLoveDogs - [seu nome]` (ex: `WeLoveDogs - Jose`)
8. **Esteja preparado** para apresentar seu trabalho na próxima RN
